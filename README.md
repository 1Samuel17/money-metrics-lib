# README - Money Metrics 

## Long Story Short

- cash flow management library
- high level view of financials for steering cash flow
- backbone library for cli tool (money-metrics-cli)

## Long Story

### Description:

This library is the backbone for the CLI tool of the same name (i.e. money-metrics-cli) which serves the purpose of providing a "bird's eye" view (in numbers) of personal cash flow data points (such as money coming in, money going out, money leftover, money needed, etc). 

The intent is _not_ to model after current money management/budgeting apps (which go in depth into financial data points such as transaction histories, receipt collecting, spending categories, etc). But instead, "zoom out" to a bird's eye view of cash flow to steer financial decision making.

Why this approach?

I've personally tried many money management/budgeting apps and software and they all having very good financial features and functionalities. However, for me personally, I do not need to see spending broken down into categories and subcategories to help me make financial decisions. Nor do I need waste time taking receipt photos and entering the receipt data. All of these features are great for businesses, accountants and tax professionals that need to examine every penny, but for me and financial decision making, I only need to see the following things:
- What money is coming in and from where
- Where money is going out and when
- How much money is leftover and
- If this financial situation remains the same (or changes), what can i expect in the future (hypothetical scenarios and future projections)

These core data points make it easy to steer my cashflow in a simplified way. For example, if my bird's eye view reveals that I will only have $100 leftover for the month, that tells me that I need to try to work more overtime, re-examine financial obligations for potential changes or supplement my income in some way. I don't need to see the fine-grain details of spending. I just need to see these data points to steer my cashflow strategically.
