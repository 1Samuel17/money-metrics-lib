# README - Money Metrics

- cash flow management library

## Long Story

### Description:

This library is the backbone for a CLI tool of the same name (i.e. money-metrics-cli) which serves the purpose of providing a high level overview (in numbers) of personal cash flow (such as money coming in, money going out, money leftover, money short, etc). 

The intent is _not_ to model after current money management/budgeting apps (which go in depth into financial data points such as transaction histories, receipt collecting, spending categories, etc). But instead, "zoom out" to a high level view of cashflow to steer financial decision making.

Why this approach?

I've personally tried many money management/budgeting apps and software and they all having very good financial features and functionalities. However, for me personally, I do not need to see spending broken down into categories and subcategories to help me make financial decisions. Nor do I need waste time taking receipt photos and entering the receipt data. All of these features are great for businesses, accountants and tax professionals that need to examine every penny, but for me and financial decision making, I only need to see the following things:
- What money is coming in, when and from where
- Where money is going out, when and to where
- How much money is leftover or short
- If this financial situation remains the same (or changes), what can i expect in the future (hypothetical scenarios and future projections)

These core data points make it easy to steer my cashflow in a simplified way. For example, if I see that I will only have $100 leftover for the month, that tells me that I need to try to work more overtime, re-examine financial obligations for potential changes or supplement my income in some way. I don't need to see the fine-grain details of spending. I just need to see these data points to steer my cashflow strategically.
